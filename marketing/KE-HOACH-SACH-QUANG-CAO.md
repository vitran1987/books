## Thông tin dự án Bạn Giỏi (bangioi.vn)

- **Nền tảng**: Giáo dục online tận dụng AI cho thị trường Việt Nam
- **Tuổi đời**: 1 năm (startup giai đoạn đầu)
- **Trạng thái hiện tại**: 
  - MVP hoàn thành
  - 200+ users đăng ký
  - 60,000 visits/tháng
  - Hỗ trợ môn Toán (lớp 1-12)
  - Dự kiến đầy đủ các môn học vào tháng 3/2026

- **Tính năng**: 
  - Hướng dẫn lý thuyết
  - Bài tập thực hành
  - Đề luyện thi
  - Tài liệu học và dạy
  - Công cụ AI cá nhân hóa
  - Công cụ quản lý lớp

- **Mô hình kinh doanh đang nghiên cứu**:
  1. Freemium model (học sinh không có tiền nhưng muốn học trước)
  2. Thu nhập từ quảng cáo (ưu tiên)
  3. Bán các chức năng nâng cao
  4. Crypto token (utility token - kế hoạch dài hạn)

- **Trạng thái quảng cáo**: Google Adsense đã kích hoạt 3 tuần, hiệu quả hạn chế

## PHƯƠNG PHÁP VIẾT (Writing Methodology)

### Content Style:
- ✅ **High-level technical guidance** (methods, strategies, approaches)
- ✅ **Architecture and solution comparison** (pros/cons, trade-offs)
- ✅ **Decision frameworks** (when to use what)
- ✅ **Conceptual understanding** (how things work at concept level)
- ❌ NOT: Low-level code examples
- ❌ NOT: Detailed configuration settings
- ❌ NOT: Step-by-step coding tutorials

### Example Approach:
**GOOD**: "Header bidding là phương pháp cho phép nhiều ad networks đấu giá đồng thời, thay vì waterfall tuần tự. Trade-off: higher CPM nhưng phức tạp hơn để implement và maintain. Phù hợp khi traffic > 500k/month."

**NOT GOOD**: "Step 1: Install prebid.js library. Step 2: Configure bidders in config.js file with bidder params..."

### Focus Areas:
1. **WHY** (tại sao cần phương pháp này)
2. **WHAT** (phương pháp là gì, concept)
3. **WHEN** (khi nào nên dùng)
4. **TRADE-OFFS** (pros/cons, so sánh với alternatives)
5. **DECISION FRAMEWORK** (làm sao quyết định có dùng không)

### NOT Focus:
1. ~~HOW (step-by-step code implementation)~~
2. ~~Detailed configuration values~~
3. ~~Specific API endpoints và parameters~~
